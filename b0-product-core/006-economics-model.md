# 006 — Product Economics (P&L Model)

## 1. Purpose of the Layer

The **Product Economics (P&L) layer** defines how the product generates revenue, incurs costs, and produces financial outcomes.

It describes:

- revenue streams
- cost structures
- cost drivers
- profit calculation logic
- economic relationships between product elements

It answers:

- How does the product make money?
- Where do costs come from?
- What drives revenue and expenses?
- What is the economic impact of product behavior?

---

## 2. Layer Role in AISMM

This layer connects:

```text
Value → Money
```

It translates value creation into economic results.

Other layers rely on it to:

- evaluate product viability
- optimize cost vs value
- prioritize investments
- assess profitability

This layer does NOT define:

- business strategy
- accounting details
- financial reporting formats

It defines **product-level economic logic**.

---

## 3. Main Output of the Layer

The output is a structured economic model:

- revenue streams
- cost categories
- cost drivers
- profit formulas
- mappings to product behavior and value

---

## 4. Core Concepts

### 4.1 Revenue

Revenue is income generated by the product.

Examples:

```text
revenue.subscription
revenue.transaction_fee
revenue.usage_fee
```

---

### 4.2 Cost

Cost is any resource consumption required to operate or develop the product.

Examples:

```text
cost.infrastructure
cost.sms
cost.team
```

---

### 4.3 Cost Driver

A cost driver is an element that causes cost.

Examples:

- API call
- workflow execution
- data storage
- user activity

---

### 4.4 Revenue Driver

A revenue driver is an element that generates revenue.

Examples:

- subscription activation
- transaction
- usage event

---

### 4.5 Profit

Profit is defined as:

```text
profit = revenue - cost
```

---

### 4.6 Economic Unit

Defines how value is measured economically.

Examples:

- per user
- per transaction
- per workflow execution

---

### 6.5 Profit Logic

Define how profit is calculated.

---

### 6.6 Unit Economics

Define economic units:

- per user
- per action
- per workflow

## 7. Preferred Representation

The semantic content of this layer is independent of any specific representation format.

This layer defines **economic relationships and calculations**, not how they must be stored.

Due to its structured nature, the following representations are suitable:

- Markdown (`.md`) for explanations and structure  
- Structured formats (JSON / YAML) for machine-readable economic models  
- Tables for financial clarity  

Implementations:

- SHOULD use structured formats for calculations  
- SHOULD use Markdown for explanation  
- MAY use other formats if semantics are preserved  
- MUST NOT encode meaning in format-specific constructs  

The correctness of this layer is determined by the completeness and consistency of the economic model.
