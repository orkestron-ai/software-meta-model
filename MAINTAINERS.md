# Maintainers

This file lists the people responsible for AISMM. The decision-making process
that maintainers follow is described in [`GOVERNANCE.md`](./GOVERNANCE.md), and
the contribution process is in [`CONTRIBUTING.md`](./CONTRIBUTING.md).

| Maintainer       | GitHub          | Role            | Areas |
| ---------------- | --------------- | --------------- | ----- |
| Victor Bolshakov | @orkestron-ai   | Lead maintainer | Overall spec direction, versioning, governance |

> **Note:** add additional maintainers as the project grows. Each maintainer
> should have a GitHub handle and one or more areas of responsibility (for
> example specific bundles `b0`–`b12`, schemas, or governance tooling).

## What maintainers do

- Triage issues and change requests (Accepted in principle / Needs discussion /
  Out of scope), as described in [`CONTRIBUTING.md`](./CONTRIBUTING.md).
- Review and merge pull requests; ensure schema and spec stay in sync.
- Own versioning decisions per
  [`aismm-versioning-and-conformance.md`](./aismm-versioning-and-conformance.md)
  and cut releases per [`RELEASING.md`](./RELEASING.md).
- Uphold the [`CODE_OF_CONDUCT.md`](./CODE_OF_CONDUCT.md).

## Becoming a maintainer

Contributors who have a sustained track record of high-quality, accepted
contributions and constructive review may be nominated by an existing
maintainer. Nominations and the addition/removal of maintainers follow the
process in [`GOVERNANCE.md`](./GOVERNANCE.md).
